# 기준 산정 모델 (2026-06)

**마지막 갱신:** 2026-06-05  
**상태:** 베타 운영·UI 반영. 유료 슬롯 숫자는 [`PRD.md`](PRD.md) 참고.

---

## 1. 한 줄

프로젝트 안에서 **맞춤법·일관성은 각각 기준 1묶음**이고, 일관성 탭에서 **등록하는 검색 1건 = 기준 1개**로 센다.

---

## 2. 프로젝트(저장 단위)

| 단위 | 베타(일반) | 설명 |
|------|------------|------|
| **프로젝트** | 1개 저장 | 이름 붙인 기준 묶음 전체 (`savedAt` 프리셋) |

원고·PDF는 저장하지 않음. [`product-spine.md`](product-spine.md) 프로젝트 저장 정책.

---

## 3. 검수 기준 항목 (프로젝트 내부)

| 항목 | 개수 | 설명 |
|------|------|------|
| **맞춤법 기준** | **1묶음** | 맞춤법 확인 + 편집자 검토(주의) 토글 전체. 세부 규칙 수와 별개로 「맞춤법 기준」1개로 본다. |
| **일관성 기준** | **1묶음** | 일관성 탭 설정 전체(검색·제외·본용언+보조용언). |

---

## 4. 일관성 탭 — 검색 등록

| 하위 항목 | 등록 상한 (베타) | 비고 |
|-----------|------------------|------|
| **일관성 찾기** (문자열) | **기준 슬롯 5개** | 쉼표로 넣은 **항목마다 슬롯 1개** (예: `a,b,c` = 3슬롯) |
| **공통 문자열 찾기** (`@`) | **5개** | `@` 패턴 1개 = 기준 1개. 일관성 찾기 슬롯과 별도 |
| 검사 제외 문구 | (상한 별도) | 검색 기준과 별도 |
| 본용언+보조용언 | 시트 시드 | 개발중 · 일관성 검사 제외 항목 |

코드(베타 상한, 하드코딩): `MAX_CONSISTENCY_CRITERIA_SLOTS` (= 5), `MAX_PHRASE_SLOT_REGISTERED_ENTRIES` (= 5).

---

## 5. 「기준 슬롯」이란 (유료화 축)

**기준 슬롯** = 프로젝트 안에서 **일관성 검색 등록 1건이 차지하는 사용량 단위**.

| 예 | 슬롯 소비 |
|----|-----------|
| 일관성 찾기 `조선˅시대,조선시대` 한 번에 등록 | 쉼표 항목 수만큼 (2슬롯) |
| 공통 문자열 `@시대` 1개 등록 | 1슬롯 (일관성 찾기 슬롯 풀과 **별도 카운트** 가능) |

맞춤법 기준·일관성 기준 **묶음** 자체는 슬롯이 아니라 **프로젝트에 들어 있는 설정 덩어리**다.  
슬롯은 **「얼마나 많이 찾을 패턴을 등록·돌릴 수 있나」** 를 나중에 제한할 때 쓰는 **미터**다.

### 베타 (지금)

- 상한 **5+5** 등은 코드에 고정. **결제·플랜과 연동하지 않음.**
- 프로젝트 저장 **1개**, 일일 검수 쿼터만 별도(`betaDailyQuota`).

### 유료 (예정) — **프로젝트 개수 + 기준 슬롯**

| 축 | 의미 | 예 (합의 초안) |
|----|------|----------------|
| **프로젝트 슬롯** | 저장 가능한 **프로젝트(기준 묶음) 개수** | 1 / 2 / 3 |
| **기준 슬롯** | (프로젝트당 또는 계정당) **일관성 검색 등록·사용 한도** | 플랜별 숫자 — 베타 5를 기본선으로 조정 |
| **검수 슬롯** | KST **하루 전체 검수 횟수** | 1 / 3~5 / 5~10 |

구현 시: `criteriaPresetLimit` · `consistencyRuleLimit` · `betaDailyQuota` → **플랜 테이블 한곳**에서 `projectSlots` + `criteriaSlots` + `dailyCheckSlots` 읽기.

**지금은 문서·카운트 규칙만 맞춰 두고, 유료 게이트는 넣지 않는다.**

## 6. UI·문구

- 맞춤법 패널: `맞춤법 기준` (묶음)  
- 일관성: `일관성 찾기(기준 슬롯 5개)` · `공통 문자열 찾기(5개)`  
- 저장 알림: `「이름」 프로젝트가 저장되었습니다.`  
- 삭제 확인: `「이름」 프로젝트를 삭제할까요?`
