# Product Spine — PDF 출판 교정 보조 툴

**출판·교정 실무용 PDF 검수 보조**의 기획·기술·범위 **단일 뼈대 문서**입니다.  
앱은 이 파일을 **읽지 않습니다**. 개발·AI·검수 시 **결정이 어긋나지 않게** 하는 용도입니다.

**마지막 갱신:** 2026-05-22 (모모 마스코트·피드백 반영)

---

## 한 줄 정의

> **AI 문장 교정이 아니라**, 조판 PDF에서 **반복 검수(맞춤법·표기 일관성)를 자동 탐지**하고, **PDF 위에 위치를 표시**하는 **브라우저 로컬** 도구.

---

| 원칙 | 내용 |
|------|------|
| 자동 수정 | ❌ |
| 자동 탐지 | ⭕ |
| 처리 위치 | 브라우저 로컬 (PDF.js) |

| 디자인 | 최소, 기능 우선 |

**포지셔닝:** 맞춤법 앱 대체 X → **「PDF 교열 전처리 스캐너」**

## 대상 사용자

- 출판사 편집자 · 교정·교열자 · 번역 편집자 · 인디 출판 제작자

**Pain Point (왜 로컬인가)**

- 보안·원고 유출 우려 → 서버 업로드 회피
- 같은 규칙·같은 오탈자 **반복 검수 피로** → 규칙 기반 일괄 탐지

1. PDF 업로드  
2. PDF 텍스트 추출  
3. 맞춤법 규칙 검사 (내장)  
4. 사용자 일관성 규칙  
5. 다중 규칙 동시 검사  
6. 발견 위치 표시 (리스트)

8. 클릭 시 해당 위치·페이지 이동  

**화면:** **대문(웰컴)** · 검수 화면 좌측 탭 **맞춤법·일관성** · 탭 왼쪽 **대문(책)** 버튼 · **피드백**은 대문만

| 항목 | 결정 |
|------|------|
| 규칙 수 | 활성 규칙 **최대 1000개** (내장 맞춤법+일관성·개별 규칙 합산) |
| 하이라이트 | **PDF.js TextLayer DOM span + CSS** (좌표 직접 역산 X) |

| Web Worker | **제외** |

**개발 순서 (리스크 순)**

1. 샘플 PDF 2–3종 POC: 본문 / 각주 / 표·다단 → TextLayer 매칭·하이라이트  
2. 공백 유연 find + 결과 **인스턴스 단위** (`규칙 | p.N (k건)`)  
3. 규칙 CRUD · localStorage · 리스트 클릭 → 페이지 스크롤  
4. 맞춤법 내장 JSON + 일관성 사용자 규칙  

**v1.0 완료 정의 (요약)**

- [x] PDF 업로드 후 텍스트 선택 가능한 PDF에서 검사 동작 (v0.1)

- [ ] find에 공백 있는 규칙이 조판 PDF 줄바꿈에서도 **합리적** 탐지  
- [x] 새로고침 시 **IndexedDB**로 PDF·검사 결과·페이지 복원 (이 브라우저만 · 「작업 지우기」로 삭제)  
- [ ] 서버·다른 PC로 PDF/결과 전송 없음

## 규칙 시스템

### 형태 (JSON)

```json
{
  "name": "경제 출판사 스타일",
  "rules": [
    { "find": "우리 나라", "replace": "우리나라", "enabled": true }
  ]
}
```

- **내장 맞춤법:** Google 시트 `spelling_rules` → `npm run sync-spelling` → `src/data/spelling-rules.json`  
- **일관성:** 사용자 등록 · ON/OFF · **맞춤법 탭과 검사 분리**(일관성 탭에서만 일관성 규칙 검사)  
- **일관성 등록 (표기 찾기 한 박스):**  
  - **문자열 찾기** — 등록한 그대로 (예: `조선˅시대`, `RED˅PEN`)  
  - **공통 문자열 찾기** — `@` 자리 (예: `@시대` 붙임 / `@˅시대` 띄움)  
  - **검사 제외 문구**  
  - **본용언+보조용언 띄어쓰기** — `bon-bojo` 시트 sync · 전체 선택 체크 · **개발중**(규칙 수·일관성 검사 제외)  
- **띄어쓰기 검토(맞춤법 탭 하단):** 체크한 항목만 PDF 표시 · 붙임/띄움은 사용자 판단  

### find → 검사용 regex (v1.0 필수)

- 사용자 입력은 **일반 문자열** 유지  
- 엔진 내부: 공백·줄바꿈·NBSP → `\s+` 등으로 변환  
- **과유연 오탐** 주의 — 샘플 PDF로 검증  

### 검사 흐름

```text
PDF 업로드 (메모리만)
→ 텍스트 추출 (페이지별)
→ 활성 규칙 로드
→ 페이지별 탐색 (청크)
→ 인스턴스 목록 { ruleId, page, spanRefs[] }
→ TextLayer span에 highlight class
→ 리스트 클릭 → 해당 페이지 스크롤 + 강조
```

**Worker 분리 (v1.1):** 검사·추출 → Worker / DOM 하이라이트 → 메인 스레드

## 프로젝트 저장 정책 (「프로젝트」= 규칙 세트)

UI 라벨 권장: **「규칙 세트」** 또는 **「스타일 프리셋」** — PDF 프로젝트가 아님을 명시.

### 저장됨 ✅

- 규칙 세트 **이름** (예: `경제 출판사 스타일`)  
- **규칙 목록** (활성 최대 1000개)  

### 저장 안 됨 ❌

- PDF 파일  
- 검사 결과 · 검사 히스토리  
- 어떤 PDF를 검사했는지 (파일명·해시 포함 **기록 없음**)  
- **피드백 본문** (앱 localStorage/IndexedDB에 보관하지 않음)  

### 세션 vs 디스크 (UI 한 줄)

> 규칙만 저장됩니다. PDF와 이번 검사 결과는 브라우저를 닫으면 사라집니다.

### v1.1 UX (정책 유지, 선택)

- 규칙 세트 **JSON 가져오기/보내기** (PDF·결과 없음)  
- **수동** 검사 결과 CSV 다운로드 (자동 저장 X)

## 피드백 (v0.1+)

| 항목 | 결정 |

| 진입 | **대문(웰컴) 화면** CTA 영역 **피드백 보내기** → **모달** (검수 화면 헤더에는 없음) |

| 앱 저장 | ❌ 피드백 텍스트는 앱에 남기지 않음 |

### Form 미연결 (기본)

- **보내기** → 분류+내용을 **클립보드에 복사** (개발·운영자가 Form 붙이기 전에도 의견 수집 가능)  
- 모달에 「Google Form 연결 전」 안내 표시  

### Google Form 연결 (운영 시 `.env`)

```env
VITE_FEEDBACK_FORM_ACTION_URL=   # …/formResponse
VITE_FEEDBACK_FORM_ENTRY_TYPE=    # entry.xxxxx (분류 필드)
VITE_FEEDBACK_FORM_ENTRY_MESSAGE= # entry.yyyyy (본문 필드)
```

- 설정 후 **보내기** → Form에 `POST` (`no-cors`, 브라우저에서 직접)  
- **원문·PDF는 전송하지 않음** — 사용자가 적은 피드백만 Form으로 감  
- Form·entry ID 변경 시 `.env`만 갱신 (코드 배포 없이 조정 가능)  
- **`no-cors`라 앱은 수신 확인 불가** → 연결 후 Form **응답 탭**에서 1회 검증 필수
