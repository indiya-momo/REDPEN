**단일 뼈대:** [`product-spine.md`](product-spine.md) · **전략 메모:** [`strategy-decisions-2026-06.md`](strategy-decisions-2026-06.md)

## 1. 한 줄 정의

> **AI 문장 교정이 아니라**, 조판·텍스트 PDF에서 **맞춤법·표기 일관성을 자동 탐지**하고 **PDF 위에 위치를 표시**하는 **브라우저 로컬** 검수 도구.

**포지셔닝:** 맞춤법 앱 대체 X → **PDF 교열 전처리 스캐너** · 장기적으로 **작품별 표기 기준(프로젝트) 관리**.

---

## 2. 문제 (Pain Point)

| 사용자 | 문제 |
|--------|------|
| 출판 편집·교정 | 같은 규칙·같은 오탈자 **반복 검수** 피로 |
| 번역·기획 편집 | 원고 유출 우려 → **서버 업로드 회피** |
| (장기) 웹소설·시나리오 | 시리즈·권 전체 **표기 A/B 통일** — 맞춤법 앱과 다른 니즈 |

---

## 3. 핵심 원칙 (변경 없음)

| 원칙 | 내용 |
|------|------|
| 자동 수정 | ❌ |
| 자동 탐지 | ⭕ |
| 처리 위치 | 브라우저 로컬 (PDF.js) |
| **원고·PDF 본문** | **서버 저장 금지** |
| **클라우드 저장** | **프로젝트(기준 묶음)만** |
| AI 문장 교정 | MVP·베타 비목표 |

## 4. 대상 사용자 (단계별)

### 4.1 지금 (오픈 베타 ~1달)

- 출판사 편집자 · 교정·교열자 · 인디 출판 제작자  
- 빨간펜 등 **편집자 커뮤니티** 베타 테스터  

### 4.2 이후 (유료 + 마이페이지 시연 가능 후)

- 웹소설 작가 · 시나리오 작가 (파일럿)  
- 번역가 · 번역 편집 (파일럿)  

**주의:** 베타 직후 웹소설·번역 **대량 영업하지 않음** — 출판 베타에서 제품·가격 검증 우선.

## 5. 기능 요구사항

### 5.1 v1.0 / 오픈 베타 (구현·운영 중)

| # | 기능 | 상태 |
|---|------|------|
| 1 | PDF 업로드 (메모리·IndexedDB 세션) | ✅ |
| 2 | PDF 텍스트 추출 (Hancom PDF 부분 보정) | ✅ |
| 3 | 맞춤법·주의 규칙 검사 (시트 sync) | ✅ |
| 4 | 사용자 일관성 규칙 (쉼표 일괄·`@` 패턴) | ✅ |
| 5 | 다중 규칙 동시 검사 | ✅ |
| 6 | 결과 리스트 + PDF 하이라이트 | ✅ |
| 7 | 클릭 시 페이지 이동 | ✅ |
| 8 | **프로젝트**(규칙 세트) 저장 — 계정당 1개 (일반) | ✅ |
| 9 | Firestore 기준 클라우드 동기화 | ✅ |
| 10 | 로그인·일일 검수 쿼터 (베타) | ✅ |
| 11 | 파일–원고 페이지 보정 | ✅ |
| 12 | 본용언+보조용언 (개발중, 검사 제한적) | 🚧 |

**화면:** PC 대문 · 검수 화면(맞춤법/일관성 탭) · 마이페이지 · 모모의 방

### 5.2 베타 보완 (~1달)

- 베타 피드백 기반 UX·안내·막힘 지점 수정  
- PostHog: `check_run`, 저장, 이탈  
- 한글에서 저장한 **PDF 샘플** 품질 이슈 수집 ([`input-format-strategy.md`](input-format-strategy.md))  
- 주의 규칙 `except` 등 데이터 보정 (매칭 로직 변경은 freeze 하에)

### 5.3 유료화 (합의 방향)

**플랜 예시 (숫자 조정 가능):**

| | 무료/베타 | 7,900원 | 10,900원 팀 |
|--|-----------|---------|-------------|
| 프로젝트 슬롯 | 1 | 2 | 3 |
| 검수/일 (KST) | 1 | 3~5 | 5~10 |
| 마이페이지 프로젝트 목록·세부 | 기본 | ✅ | ✅ |
| 공유 링크 **발급** | ❌ | ❌ | ✅ |
| 링크 **가져오기** | ✅ | ✅ | ✅ |

**슬롯:** 플랜별 `projectSlots` + `dailyCheckSlots` 단일 테이블로 관리 (구현 예정).

**공유:** 실시간 협업 없음. 링크 = 읽기 전용 스냅샷 → 수신자 프로젝트 슬롯에 **복사**.

### 5.4 마이페이지·프로젝트 (유료 전 필수)

- [ ] 프로젝트 목록·이름·저장일  
- [ ] 규칙 요약(활성 수 등)  
- [ ] (팀) 공유 링크 발급·가져오기  
- [ ] 결제·플랜 표시  

→ **웹소설·번역 타진은 이 화면이 시연 가능한 뒤.**

### 5.5 Post-beta — 엔티티·클러스터 (로드맵)

**목표:** 수동 일관성 등록 전에 **반복 이름 후보 자동 묶기** → 대표 표기 확정 → 기존 검수 엔진.

| 층 | 내용 | 위치 |
|----|------|------|
| 입력 | PDF/텍스트, 프로젝트 선택 | 프론트 |
| 분석 | 빈도·정규화·클러스터 | **로컬 90%** |
| AI | 애매 클러스터만 | **10% 이하** |
| 저장 | rules(canonical, aliases) | Firestore — **문서 본문 없음** |

**1차 제외:** 실시간 협업, 완전 자동 교정, 서버 원문, 일괄 치환(미리보기만 1차).

상세: [`strategy-decisions-2026-06.md`](strategy-decisions-2026-06.md) §6.

### 5.6 Post-beta — 입력 형식

| 우선순위 | 형식 | 시기 |
|----------|------|------|
| 1 | PDF + **한글 PDF** 품질 | 베타~유료 |
| 2 | HWPX POC (텍스트 검수, 하이라이트 2차) | 타진 전 결정 |
| 3 | TXT (번역) | 필요 시 |

상세: [`input-format-strategy.md`](input-format-strategy.md).

## 6. 비목표

### MVP·베타 (유지)

- AI 자동 교열 · 문장 추천  
- 자동 수정 저장 · PDF 재생성  
- OCR · 스캔 PDF 작업  
- 서버에 PDF·검수 결과·**원고 본문** 저장  
- 실시간 협업 · 공동 편집  

### 베타 freeze (무단 변경 금지)

- `ruleEngine` · regex · spelling/caution 매칭 로직  
- `MainScreen` dead props 대규모 제거  

### Post-beta 검토 (즉시 개발 X)

- HWP/HWPX 전면 지원  
- 엔티티 NLP 명사 추출 (초기는 빈도·문자열 기반)

## 7. 데이터 모델 (요약)

### 로컬

- PDF 바이너리 (IndexedDB 세션)  
- 검수 결과 (세션)  
- (장기) 클러스터 분석 중간값  

### 서버 (Firestore)

- 사용자 프로필  
- `ruleSets` + `activeSetId` (프로젝트)  
- (장기) 팀 공유 **규칙 스냅샷** — 토큰 URL  
- `betaDailyUsage` (검수 쿼터)  

### 저장하지 않음

- PDF 파일 · 원고 텍스트 전체 · 검수 히스토리 영구 보관
