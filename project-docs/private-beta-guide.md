# 비공개 베타 참여 가이드 (REDPEN)

**대상:** 초대받은 베타 테스터  
**앱 버전:** `v0.7.0` (화면 하단·대문 푸터에서 빌드 날짜·ID 확인)  
**마지막 갱신:** 2026-05-30  
**성격:** 제한적 **비공개** 베타 — 기능·안정성은 개선 중입니다.

> **한 줄 정의:** AI 교정기가 아니라, **조판 PDF에서 맞춤법·표기 일관성을 자동 탐지**하고 **PDF 위에 위치를 표시**하는 **브라우저 로컬** 도구입니다.

## 1. 시작하기

### 접속

- 배포 URL은 **초대 메일·운영자**가 안내한 주소를 사용하세요.
- GitHub Pages 배포 시 저장소 `indiya-momo/REDPEN` 기준 경로는 운영자가 `/docs` 루트로 설정합니다.
- **버전 확인:** 대문 또는 검수 화면 좌하단 **버전 뱃지** (`v0.1.0 · 날짜 · 빌드 ID`)와 안내 URL이 일치하는지 확인하세요.

### 첫 실행 체크

- [ ] Chrome 또는 Edge **최신 버전** (권장)
- [ ] **텍스트가 선택되는 PDF** (스캔본·이미지만 있는 PDF는 불가)
- [ ] **50MB 이하** PDF (초과 시 업로드 단계에서 안내)
- [ ] 대문 하단 **통계 수집** 문구 확인 — 원하면 「수집 안 함」 가능

## 2. 이 앱이 하는 일 / 하지 않는 일

| ⭕ 하는 일 | ❌ 하지 않는 일 |
|-----------|----------------|
| PDF 텍스트 추출 후 규칙 기반 **탐지** | 문장 **자동 수정**·AI **교정 추천** |
| 맞춤법·띄어쓰기·일관성 **하이라이트** | PDF **재생성·저장** |
| 클릭 시 해당 **페이지·위치 이동** | HWP, OCR, **스캔 PDF** |
| 브라우저 **로컬** 처리 (원고 서버 업로드 없음) | 로그인·협업·클라우드 원고 보관 |

## 3. 권장 환경

| 항목 | 권장 |
|------|------|
| OS | Windows 10/11, macOS (최신) |
| 브라우저 | **Google Chrome**, **Microsoft Edge** |
| Safari / Firefox | 동작은 가능하나 **PDF 열기·세션 복원**이 불안정할 수 있음 |
| 네트워크 | 최초 로드·피드백 전송 시 필요. **검수 자체는 오프라인 가능**(이미 연 PDF) |

**「PDF 열기」** 버튼(File System Access)을 쓰면 새로고침 후 **같은 파일을 다시 연결**하기 쉽습니다. 드래그·파일 선택만 쓰면 세션 복원이 제한될 수 있습니다.

## 4. PDF 조건

| 조건 | 설명 |
|------|------|
| 형식 | **텍스트 PDF** (인디자인 등에서 내보낸 PDF) |
| 스캔 PDF | **지원하지 않음** — 텍스트 선택·복사가 안 되면 사용 불가 |
| 용량 | **50MB 이하** (REDPEN 운영 한도 — 브라우저 기술 한도와 별개) |
| 페이지 | **약 300페이지 이하 권장** (신국판 기준 가이드) |
| 용량 초과 | PDF **나누기** 안내 링크가 업로드 화면에 표시될 수 있음 |

## 5. 기본 사용 흐름

1. **대문** → 「검수 시작」
2. **PDF 열기** (또는 파일 선택)
3. 좌측 탭에서 **맞춤법 확인** / **일관성 확인** 선택
4. 적용할 **내장 규칙·주의 규칙·일관성 규칙** 켜기/끄기
5. 중앙 **「검수 실행」** → 완료 후 좌측 **결과 목록** + 우측 **PDF 하이라이트**
6. 결과 항목 클릭 → 해당 페이지·위치로 이동
7. **작업 종료** → 대문으로 (PDF·검사 상태는 세션 정책에 따름)
8. 문제·제안 → 우측 상단 **「피드백 보내기」**

## 6. 저장되는 것 / 저장되지 않는 것

### 이 기기(브라우저)에 남는 것

| 데이터 | 저장 위치 | 언제 |
|--------|-----------|------|
| 맞춤법·주의 **토글 ON/OFF** | localStorage | 규칙 변경 후 **자동 저장** (약 0.4초 debounce) |
| **일관성 규칙**·제외 구문 | localStorage | 편집 시 자동 저장 |
| **PDF + 검사 결과** (세션) | IndexedDB 등 | 검사·페이지 이동 후 **자동 저장 시도** |

### 서버로 가지 **않는** 것

- PDF **파일 내용**·원고 **텍스트**
- 규칙 find/replace **문자열**
- 검사 **매칭 결과** 본문

### UI와 다른 점 (중요)

- 검수 화면에 **「사용자의 기준을 저장하는 기능을 준비 중입니다」** 가 보입니다.
- **규칙 세트 이름·여러 세트 관리·「저장」 버튼** UI는 **아직 없습니다**.
- 그러나 **토글·일관성 규칙은 이미 자동 저장**됩니다. 「저장 버튼이 없어서 설정이 날아간다」고 느끼면 **아래 FAQ**를 참고하세요.

## 7. 알려진 제한 (Known Issues)

베타 중 **정상으로 간주하거나**, **피드백을 부탁드리는** 항목입니다.

| # | 현상 | 설명·대처 |
|---|------|-----------|
| 1 | **새로고침 후 PDF 없음** | 세션 복원이 브라우저·권한·용량에 따라 실패할 수 있음. **같은 PDF를 다시 열기**. 「PDF 열기」 사용 권장 |
| 2 | **파일 접근 권한 요청** | 이전에 연 PDF에 다시 접근 허용 필요할 수 있음 |
| 3 | **「기준 저장 준비 중」 문구** | 규칙 **세트 UI** 미구현. **토글 autosave는 동작** |
| 4 | **규칙 1000개 한도** | 활성 규칙 합계 **1000개 초과** 시 alert — 줄인 뒤 재시도 (UI에 슬롯 개념은 비노출) |
| 5 | **대용량·다규칙 검사 느림** | 진행률 표시·검사 중 버튼 비활성 — 완료까지 대기 |
| 6 | **스캔 PDF** | 텍스트 없음 → 검수 불가 (의도된 거절) |
| 7 | **피드백 「보냈습니다」** | Google Form **실제 수신은 Form 응답 탭**에서만 확인 가능 (앱은 no-cors POST) |
| 8 | **모바일** | PC 우선. 모바일은 **대문·모모룸** 위주 — **본격 검수는 PC 권장** |

**개발팀 내부 메모 (테스터 참고용):** 세션·규칙 세트 계약 상세는 `project-docs/app-mainscreen-contract.md`, 구조 리스크는 `project-docs/post-stabilization-audit-report.md`.

## 8. 개인정보·통계 수집

### PostHog (익명 이용 통계)

- **수집:** 세션 시작, PDF 열기(페이지·용량 **구간만**), 검사 실행, 결과 조회, 피드백 모달 열기 등 **6종 이벤트**
- **수집 안 함:** PDF 파일명·본문, 규칙 문자열, 피드백 **본문**, Session replay, autocapture
- **거부:** 대문 하단 **「수집 안 함」** → `localStorage` opt-out (같은 브라우저)

### Google Form (피드백)

- 피드백 **텍스트**는 Form 제출 시 Google 측으로 전송됩니다.
- 운영자만 응답 스프레드시트에서 확인합니다.

자세한 운영 설정: `project-docs/analytics-beta.md`

## 10. 자주 묻는 질문 (FAQ)

**Q. 설정이 저장이 안 되는 것 같아요.**  
A. **토글·일관성 규칙은 자동 저장**됩니다. 브라우저 데이터 삭제·시크릿 모드·다른 PC에서는 이어지지 않습니다. 「저장」 버튼이 없는 것은 UI 미완이지, autosave 미동작이 아닙니다.

**Q. 새로고침하면 PDF가 사라져요.**  
A. **알려진 제한**입니다. 같은 파일을 다시 열고, 가능하면 **「PDF 열기」**를 사용하세요.

**Q. 맞춤법 결과가 틀려요.**  
A. **탐지 도구**입니다. 최종 판단은 편집자·교정자가 합니다. 규칙 제안은 피드백으로 알려주세요.

**Q. 1000개 규칙 제한 alert가 떠요.**  
A. 켜 둔 **내장·주의·일관성 규칙 합**이 한도를 넘었습니다. 일부 규칙을 끈 뒤 다시 켜세요.

**Q. 서버에 PDF가 올라가나요?**  
A. **아니요.** 검수는 브라우저 로컬입니다. (통계·피드백만 별도 — §8 참고)
