# 주의 규칙 `match_mode` 가이드

PDF 주의(사용자 직접 판단) 규칙이 텍스트에서 **어떤 모양을 잡아 하이라이트할지** 정하는 값입니다.  
자동 수정이 아니라 **(검토)** 표시만 합니다.

관련 문서: [맞춤법·주의 시트 운영](sheet-spelling.md) · 템플릿 `templates/caution_rules.csv`

---

## 1. 전체 흐름

```
시트 match_mode + label (+ stems)
        ↓
cautionFindPattern() → regex 문자열
        ↓
PDF 페이지 텍스트에 regex 적용
        ↓
맞으면 주의 하이라이트 (교정자가 판단)
```

| 구분 | 역할 |
|------|------|
| **`label` / `stems`** | PDF에서 **실제로 찾을 글자** |
| **`match_mode`** | **어떤 띄어쓰기·붙임 모양**을 잡을지 |
| **`display_label`** | 체크박스·결과에 **보이는 이름** (`^가지`, `∧가량` 등) |
| **`enabled`** | 앱 첫 반영 시 체크 on/off (localStorage가 이후 우선) |
| **`except`** | **검사 제외 문구** (쉼표, **통째** 일치). 합성어 등 false positive를 편집자가 시트에만 적어 둠 |

- 시트에서 `match_mode`를 **비우면** `any-before`(기본값)입니다.
- `stems`에 `주,준`처럼 여러 개면 **어간마다 regex 하나**가 생기고, **활성 규칙 수**에도 그만큼 더해집니다.

---

## 2. 모드 목록 (구현 기준)

### 2.0 방향부터 정하기 (중요)

맞춤법 주의는 **「맞는 쪽」이 아니라 「잘못된 쪽」** 을 잡습니다.  
`any-before`는 **붙임·띄움을 둘 다** 잡아서, 방향이 분명한 규칙에는 맞지 않는 경우가 많습니다.

| 올바른 표기 | 잡아야 할 **오류** | `match_mode` |
|-------------|-------------------|--------------|
| **붙여** 써야 함 (조사·접사 `∧가량`) | 앞말 **띄고** 씀 (`삼 가량`) | `spaced-before` |
| **띄어** 써야 함 (의존명사 `정도`) | 앞말에 **붙여** 씀 (`오른정도` → `오른 정도`) | **`attached-before`** |

**이름 (안만):** 코드 기본값은 `attached-before`. `spaced-before`와 짝으로 읽기 쉬운 **`before-attached`**, 줄임 **`ap-attached`** 도 sync 시 같은 모드로 인식합니다. 표기를 바꿀지는 정하지 않았습니다.

**정도 예**

| 텍스트 | 판단 | 잡을지 |
|--------|------|:------:|
| 오른**정도** | 붙임 오류 | **O** |
| 오른 **정도** | 맞음 | **X** |

→ `any-before`는 둘 다 잡으므로 **정도에는 쓰지 않음**.  
→ `spaced-before`는 **맞게 띄어 쓴 것만** 잡으므로 **정도에도 쓰지 않음**.

### 2.1 공통 기호

| 기호 | 의미 |
|------|------|
| `([^\s]{2,})` | 앞에 **공백이 아닌 글자 2글자 이상** (대략 「앞말」) |
| `[ \u00A0]+` | **일반 공백** 또는 **줄바꿈 없는 공백(NBSP)** 1칸 이상 |
| `[\uAC00-\uD7A3]+` | **한글 음절** 1글자 이상 (어미·뒤 꼬리) |
| `(?!\S)` | 바로 뒤에 **붙은 글자(비공백)** 가 이어지지 않음 → 단어 끝 쪽 매칭 |

---

### 2.2 `any-before` (비움 = 기본)

**패턴 요약:** `앞말(2글자+) + 공백(0~여러) + label`

```
([^\s]{2,})\s*{label}
```

| 텍스트 | 잡힘? | 이유 |
|--------|:-----:|------|
| 한 **가지** | O | 앞말 + 공백 + 가지 |
| **산가지** | O | 앞말 `산` + 공백 0 + `가지` |
| **여름가지** | O | 합성어도 잡힘 |
| **한가지** (붙임) | O | 공백 없어도 `\s*` 때문에 잡힐 수 있음 |

**쓰는 때**

- 붙임/띄움 **양쪽 다** 후보로 넓게 보고 싶을 때
- 예외가 많아 **사람이 문맥으로 거르는** 주의 규칙

**단점**

- **방향이 정해진 규칙**(정도, 조사 붙임 등)에는 부적합 — 붙임·띄움 **한쪽만** 잡는 모드를 쓸 것

---

### 2.3 `spaced-before` (^ — 붙여야 하는데 띄어 씀)

**패턴 요약:** `앞말(2글자+) + 공백(필수) + label` (뒤에 바로 다른 글자 붙으면 제외)

```
([^\s]{2,})[ \u00A0]+{label}(?!\S)
```

| 텍스트 | 잡힘? |
|--------|:-----:|
| 두 **가지** | O |
| **산가지** | X (공백 없음) |
| **여름가지** | X |
| **한가지** (붙임) | X |

**쓰는 때**

- **붙여 써야 하는데 띄어 쓴 경우** (`명사 만`, `∧가량` → `삼 가량`)
- 조사·접사처럼 **앞말에 붙는 쪽**이 맞는 규칙

**안 잡는 것**

- **띄어 써야 하는데 붙인 경우** (`오른정도`) → **`attached-before`**

---

### 2.4 `attached-before` (∨ — 띄어야 하는데 붙여 씀)

**패턴 요약:** `앞말(1글자+) + label` 이 **한 토큰의 끝** (뒤에 글자가 더 붙으면 제외)

```
([^\s]{1,}{label})(?!\S)
```

| 텍스트 | 잡힘? |
|--------|:-----:|
| 오른**정도** | O |
| 오른 **정도** | X |
| **산가지** | O (`산`+`가지` 붙음 — 합성어는 문맥으로 제외 판단) |
| **물가지수** | X (`가지` 뒤에 `수` 등이 이어짐) |

**쓰는 때**

- **의존명사** 등 **앞말과 띄어 써야 하는** 항목 (`정도`, `한 가지`의 `가지` …)
- **붙여 쓴 오류**만 검토 후보로 올릴 때

**안 잡는 것**

- 이미 **맞게 띄어 쓴** 경우 (`오른 정도`)

`display_label`을 비우면 UI에 **`∨정도`** 형태로 표시됩니다 (`^` = 띄어 씀 오류, `∨` = 붙임 오류).

---

### 2.5 `spaced-stem`

**패턴 요약:** `앞말(2글자+) + 공백(필수) + 어간(label) + 한글 어미`

```
([^\s]{2,})[ \u00A0]+{label}[\uAC00-\uD7A3]+(?!\S)
```

| 텍스트 | 잡힘? |
|--------|:-----:|
| 살아 **있**다 | O |
| 살아 **있**었다 | O |
| 살아**있**다 | X |
| **있**다 (문장 맨 앞 등) | X (`([^\s]{2,})` 앞말 조건) |

**쓰는 때**

- **본동사·보조 어간** (`있`, `주`, `하` + `stems=주,준`)
- 「앞 용언 + **띄고** 어간 + 어미」 한 덩어리 (`맡겨 주다`, `살아 있다`)

**`spaced-before`와 차이**

| | `spaced-before` | `spaced-stem` |
|--|-----------------|---------------|
| `살아 있다` | O (`label=있다`일 때) | `label=있` → O |
| `살아 있었다` | `label=있다`면 O | `label=있` → O (어미 포함) |
| 잡는 단위 | `label` 문자열 그대로 | `label` + **뒤 한글** |

본동사 블록은 보통 **`spaced-stem` + `stems`** 조합이 맞습니다.

---

### 2.6 `fixed-phrase`

**패턴 요약:** 시트에 적은 **문구 그대로** (+ 뒤 한글 어미). `label`에 공백이 있으면 그 공백 위치도 고정.

**예:** `stems` / `label` = `해 보`

```
해[ \u00A0]+보[\uAC00-\uD7A3]+(?!\S)
```

| 텍스트 | 잡힘? |
|--------|:-----:|
| … **해 보**다 | O |
| … **해 본**다 | O (`stems`에 `해 본`도 넣었을 때) |
| 읽어 **보**다 | X (`해` 없음) |
| **해보**다 | X (공백 없음) |

**쓰는 때**

- **띄움/붙임이 정해진 표현만** 좁게 (`해 보다` / `해보다` 분리)
- `stems`에 `해 보,해보,해 본,해본`처럼 **문구 단위**로 나열

**주의**

- 앞말 2글자 조건이 **없음** → 문장 어디서든 `해 보…`만 맞으면 잡힘
- `label`이 한 글자(`보`)만이면 범위가 넓어질 수 있음 → **문구 전체**를 `label` / `stems`에 넣는 편이 안전

## 3. 시트에 넣는 값 (별칭)

`npm run sync-caution` 시 아래도 같은 모드로 인식합니다.

| 시트에 써도 됨 | 정규 `match_mode` |
|----------------|-------------------|
| **`ap-any`**, (비움), `any` | `any-before` |
| **`ap-space`**, `spaced`, `space` | `spaced-before` |
| **`ap-attach`**, `attached`, `glue`, `붙임` … | `attached-before` |
| **`spaced-stem`**, `space-stem`, `stem` | `spaced-stem` |
| `fixed`, `phrase` | `fixed-phrase` |

**시트 권장 네 가지:** `ap-any` · `ap-space` · `ap-attach` · `spaced-stem` (`space-stem`도 동일)

**별도 열 `inventory` / `track` / `variant`**

| 값 | 의미 |
|----|------|
| `TRUE` 등 | **체크박스 없음** — `^주다` 한 칸에 묶인 변이형(본, 준…) 시트 추적용 |
| `match_mode`에 `inventory` / `track` / `variant` | 위와 동일 (체크 없음) |

→ 사용자 판단용으로 **켜 두되 규칙 수에서 빼기**는 `inventory`이 아니라, 추후 **`counts_in_quota`** 열로 지원 예정.

## 4. 모드 선택 의사결정

```
붙여 써야 하는데 띄어 썼다?  (조사·접사)
  → spaced-before  (^)

띄어 써야 하는데 붙여 썼다?  (의존명사 정도, …)
  → attached-before  (∨)

앞말 + 띄고 + 어간 + 있다/주다/한다…?
  → spaced-stem (+ stems)

해 보다 / 해보다 처럼 표현이 고정?
  → fixed-phrase (+ stems에 문구)

붙임·띄움 둘 다 후보로 넓게?
  → any-before  (정도·조사 방향 규칙에는 비권장)
```

## 5. 설계 예시 (품사·띄어쓰기별)

| 항목 | 맞는 표기 | 잡는 오류 | `match_mode` |
|------|-----------|-----------|--------------|
| 만·만큼·지 (조사) | 붙임 `시간만` | 띄움 `시간 만` | `spaced-before` |
| 만·만큼·지 (의존명사) | 띄움 `먹을 만큼` | 붙임 `먹을만큼` | `attached-before` |
| 같이 (조사) | 붙임 | 띄움 | `spaced-before` |
| 같이 (부사) | 띄움 | 붙임 | `attached-before` |
| **정도** | 띄움 `오른 정도` | 붙임 **`오른정도`** | **`attached-before`** |
| 가량·쯤 (접사) | 붙임 `삼가량` | 띄움 `삼 가량` | `spaced-before` |
| 가지 (의존명사) | 띄움 `두 가지` | 붙임 `한가지` | `attached-before` |
| 본동사 (있·주·하…) | 붙임 `살아있다` | 띄움 `살아 있다` | `spaced-stem` |

**같은 글자를 한 체크·한 tip으로 묶을 때**

- `match_mode`는 **한 가지만** 걸립니다.
- 붙임/띄움 **검사 방향까지** 나누려면 **행을 나누거나**, 넓게 `any-before` + tip으로 맡깁니다.

---

## 6. `display_label`과의 관계

| `match_mode` | `display_label` 비었을 때 UI |
|--------------|------------------------------|
| `spaced-before`, `spaced-stem` | `^` + `label` (띄어 씀 오류) |
| `attached-before` | `∨` + `label` (붙임 오류) |
| `fixed-phrase` | `label` 그대로 (예: `해 보다`) |
| `any-before` | `label` 그대로 (예: `만`) |

| 열 | 넣을 값 |
|----|---------|
| `label` | 검색용 (`가량`, `정도`) |
| `display_label` | 표시용 (`∧가량`, `∨정도`) |

**`label`에 `∧가량`을 넣지 마세요** — 검색이 깨집니다.

---

## 7. 다른 열과 비교

| 열 | 역할 |
|----|------|
| `group_id` | UI 블록·tip 묶음 (`josa-uinoun`, `verb-bon` …) |
| `item_id` | 규칙 고정 키 (`josa-uinoun-man`) |
| `label` | 검색 글자 |
| `stems` | 같은 체크로 **추가 검색** (`주,준`) — 비우면 `label`만 |
| `tip` | 그룹 설명 (첫 행만 써도 forward-fill) |
| `enabled` | 처음 열 때 체크 on/off |
| **`match_mode`** | **잡는 패턴** (이 문서) |
| `display_label` | 화면 표시 이름 |
| `inventory` | 체크 없음, 변이형 추적 |
| `counts_in_quota` | (예정) `FALSE` = 규칙 1000개 한도에 **미포함** |

---

## 8. 한계 (알아두면 좋은 점)

1. **문맥(품사)은 모름** — regex만 봅니다. 「살아 보다」가 조사인지 본동사인지는 **사람이 판단**합니다.
2. **앞말 2글자** — `([^\s]{2,})` 때문에 짧은 앞맥락은 빠질 수 있습니다.
3. **주의 ≠ 맞춤법** — 잡힌다고 틀렸다는 뜻이 아니라 **검토 후보**입니다.
4. **규칙 수** — `stems` 3개 × 체크 on = regex 3개가 **활성 규칙 수**에 더해집니다.

---

## 9. 한 줄 요약

| `match_mode` | 한 줄 |
|--------------|------|
| **`any-before`** | 앞말 뒤에 label이 **붙든 띄든** 넓게 (방향 규칙엔 비권장) |
| **`spaced-before`** (`^`) | **붙여야 하는데** 앞말+**공백**+label |
| **`attached-before`** (`∨`) | **띄어야 하는데** 앞말+label **붙임** (`오른정도`) |
| **`spaced-stem`** | 앞말 + **공백** + 어간 + **한글 어미** (본동사) |
| **`fixed-phrase`** | 적어 둔 **문구 그대로** + 어미 (`해 보` / `해보` 분리) |

---

## 10. 시트 헤더 예시

```text
group_id	item_id	label	stems	tip	enabled	match_mode	display_label	inventory	except
```

반영: `npm run sync-caution` → `src/data/caution-rules.json`, `public/data/caution-rules.json` → 앱 **새로고침**.
