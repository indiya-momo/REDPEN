# 모모 궤도 애니메이션 — 작업 회고 & 에이전트 요청 가이드

모모 대문 애니메이션(펜이 원을 그리며 빨간 궤적이 남는 효과)을 만들 때 시행착오가 길었던 이유와, 앞으로 Cursor 에이전트에게 어떻게 요청하면 빠른지 정리한 문서입니다.

**관련 문서**

- [momo-assets.md](./momo-assets.md) — PNG 제작·파일 위치·현재 상수 값
- 코드: `src/components/MomoHero.jsx`, `src/components/momoOrbit.js`, `src/components/momoOrbit.test.js`

---

## 1. 왜 오래 걸렸나

### 1.1 눈으로만 맞출 수 있는 작업

펜 끝이 선에 닿는지, 원이 한 바퀴 **끝까지** 그려지는지, 시작·끝이 어색한지는 코드만으로는 확인이 어렵습니다.  
「반만 그리고 돌아온다」→「시작·끝이 어색하다」→「이제 제일 낫다」처럼 **피드백 → 수정 → 다시 확인**이 반복되면서 사이클이 길어졌습니다.

### 1.2 SVG 구현 방식이 까다로움

같은 「원」이라도 구현에 따라 결과가 달랐습니다.

| 시도 | 문제 |
|------|------|
| `path` 두 개의 호 + `large-arc` 플래그 | 땅콩(8자)형, 반만 그려짐 |
| `stroke-dasharray`로 길이만큼 그리기 | 닫힌 경로에서 끝이 안 맞음, 시작에 점처럼 보임 |
| 펜 `getPointAtLength` vs 궤적 다른 방식 | 펜과 선이 어긋남 |
| 여러 `arc` + `Z` | 길이·dash 계산 불일치 |

**최종 안정 패턴** (현재 코드):

- 궤적: `<circle>` + `pathLength` + `stroke-dashoffset`
- 펜: `orbitPoint()` — 궤적과 **같은 원·각도 공식**
- 원 stroke 시작: `rotate(45°)` 로 4시 방향과 맞춤
- 거의 다 그리면 (`TRAIL_DONE`) dash 제거 후 **완전한 원**으로 고정

### 1.3 한 번에 여러 축을 같이 바꿈

궤도 모양, 펜 각도, 크기(1.5배), 타이밍, 그리기 방식, 코드 정리·테스트가 **한 흐름 안에서** 겹쳤습니다. 각각은 작은 변경인데 서로 엮여 있어, 고칠 때마다 **다시 쓴 것처럼** 느껴질 수 있습니다.

### 1.4 참고 자료·수치가 단계적으로 맞춰짐

스크린샷·PNG만으로는 「한 프레임씩」 검증이 어렵고, 「4시·시계방향·4초+1초」 같은 말은 명확해도 **펜 팁 픽셀**은 수치(`PEN_TIP_X/Y`)로 여러 번 조정되었습니다.

### 1.5 동작 확정 후 정리·검증

화면이 맞은 뒤 **코드 정리**, **회귀 확인**, `momoOrbit.js` 분리·단위 테스트 추가가 이어졌습니다. 동작은 유지했지만 파일 구조는 또 바뀌었습니다.

## 2. 현재 확정 스펙 (요약)

| 항목 | 값 |
|------|-----|
| SVG | `viewBox="0 0 100 100"` |
| 원 중심 | (50, 48) |
| 반지름 | 39 |
| 시작 | 4시 방향 (`ORBIT_START_DEG = 45`) |
| 방향 | 시계 방향 |
| 펜 각도 | -270° 고정 (`PEN_ROTATE_DEG`) |
| 타이밍 | 4초 그리기 + 1초 멈춤 |
| 펜 크기 | 16×30, 팁 오프셋 (2.2, 26.4) |

상세·에셋 경로는 [momo-assets.md](./momo-assets.md) 참고.

**자동 검증:** `npm test` — `momoOrbit.test.js` (원 좌표·dash·한 바퀴 후 시작=끝).  
**수동 검증:** welcome 화면에서 원이 끝까지 그려지고 1초 멈춘 뒤 반복되는지 확인.

## 3. 앞으로 에이전트에게 요청하는 방법

### 3.1 맨 위에 「완료 기준」을 적기

예시:

1. **모양:** 고양이 중심 정원, 반지름 39 (또는 「지금보다 1.5배」)
2. **방향:** 4시에서 시작, 시계 방향
3. **펜:** 각도 -270° 고정, 흰 끝이 선 끝에 붙음
4. **시간:** 4초 그리기 → 1초 멈춤 → 반복
5. **끝:** 원이 **끝까지** 그려진 뒤에만 리셋 (중간에 되돌아가면 안 됨)

### 3.2 「유지 / 변경 / 금지」 나누기

| 구분 | 예 |
|------|-----|
| **유지** | 펜 각도, 4초+1초, 시계 방향 |
| **변경** | 원 반지름만 1.5배 |
| **금지** | 리팩터·파일 분리·동작 방식 변경 |

### 3.3 한 번에 하나만

| 단계 | 내용 |
|------|------|
| 1차 | 원이 끝까지 그려지게만 |
| 2차 | 크기·위치 |
| 3차 | 코드 정리 (「화면 OK 확인 후」) |

한 메시지에 「크기 + 각도 + 리팩터」가 있으면 다시 쓴 느낌이 나고 회귀 위험이 커집니다.

### 3.4 깨졌을 때는 상태를 구체적으로

- ✅ 「반만 그리고 돌아온다」「시작·끝이 끊긴다」
- ✅ 스크린샷 1장 또는 짧은 화면 녹화
- ✅ 「welcome 화면 / `MomoHero`」처럼 **어디서** 보는지

### 3.5 수치는 viewBox 기준으로

에이전트가 추측하지 않도록:

- 중심 `(50, 48)`, `r=39`
- 펜 `16×30`, 팁 `(2.2, 26.4)`
- 각도 `-270°`

### 3.6 정리·테스트 타이밍

- 「**화면 OK 확인한 다음** 정리해」
- 「정리만 하고 **동작은 바꾸지 마**」
- 「`momoOrbit.test.js` 깨지지 않게」

## 5. 구현 시 건드리지 말 것 (에이전트용)

동작이 맞은 뒤에는 아래 조합을 바꾸지 않는 것이 안전합니다.

- 궤적: `<circle>` + `stroke-dashoffset` + `TRAIL_DONE`에서 solid
- 펜: `orbitPoint()` 와 동일한 시작 각·시계 방향
- `rotate(ORBIT_START_DEG)` 와 `ORBIT_START_RAD` 는 **같은 값** (현재 45°)

바꿀 때는 `momoOrbit.test.js` 실행 후 welcome 화면을 눈으로 확인합니다.

## 6. 한 줄 요약

이 작업은 설명이 부족해서라기보다, **눈으로 맞추는 SVG 애니메이션을 여러 방식으로 시도하다 마지막에 한 패턴으로 고정**된 시간이 길었습니다.  
앞으로는 **완료 기준 + 유지/변경/금지 + 한 번에 한 가지**만 주면 훨씬 짧아집니다.
