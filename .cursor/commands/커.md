# git commit (현재 변경분)

워크스페이스: `pdf-publish-proofread` (PC worktree). **이 폴더 변경만** 커밋한다.

## 금지

- `git push` — 사용자가 따로 요청할 때만
- `git config` 변경, `--no-verify`, force push
- `.env`·시크릿 커밋
- 요청 없는 amend (pre-commit hook 실패 시 **새 커밋**)

## 절차

1. **병렬** 실행:
   - `git status`
   - `git diff` (staged + unstaged)
   - `git log -5 --oneline` (메시지 스타일 참고)

2. 변경 분석 → **why 중심** 1~2문장 커밋 메시지 초안 (한국어 또는 저장소 관례)

3. **순서대로**:
   - 관련 파일만 `git add`
   - HEREDOC으로 메시지 전달해 `git commit`
   - `git status`로 성공 확인

4. hook이 파일을 고쳤거나 커밋이 실패하면 amend 하지 말고 **원인 수정 후 새 커밋**

## 보고

- 커밋 해시·메시지
- 포함된 파일 요약 (한 줄)
- 커밋하지 않은 변경이 남았으면 짧게 알림
