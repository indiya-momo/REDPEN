# commit + deploy (GitHub Pages)

워크스페이스: `pdf-publish-proofread` (PC worktree). **이 폴더만** 커밋·푸시한다.

## 의미

`/커` + `main` push. CI(`.github/workflows/deploy-pages.yml`)가 `npm test` → `build` → `docs/` 커밋·배포.

## 금지

- `git config` 변경, `--no-verify`, force push
- `.env`·시크릿 커밋
- `docs/` 수동 수정·커밋 (CI가 갱신)
- 요청 없는 amend (hook 실패 시 **새 커밋**)

## 절차

1. **병렬**: `git status`, `git diff`, `git log -5 --oneline`

2. 변경 있으면 `/커`와 동일하게 커밋 (메시지 why 중심)

3. **배포**:
   - `git push origin main`
   - push 거절 시 pull --rebase 후 재시도 (force push 금지)

4. **확인** (가능하면):
   - `gh run list --workflow=deploy-pages.yml --limit 3`
   - 최신 run이 `completed`·`success`인지

## 보고

- 커밋 해시·메시지 (없으면 「변경 없음, push만」)
- push 결과 (`origin/main` 동기화 여부)
- CI run URL·상태 (조회했으면)
- 미커밋 untracked 파일이 남았으면 한 줄
